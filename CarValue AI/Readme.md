End‑to‑end data pipeline (collection, cleaning, feature engineering) Machine‑learning model training, evaluation, and hyper‑parameter tuning RESTful API exposing prediction endpoint Responsive, premium UI with AI‑enhanced interactions (price suggestions, chat assistance) User authentication, profile management, and saved predictions Deployment to a cloud platform with CI/CD User Review Required Dataset source: Confirm if you have a preferred dataset (e.g., Kaggle "Car Price Prediction" or a custom CSV). Project Structure Create a monorepo with frontend/ (React + Vite) and backend/ (FastAPI Python). Add data/ for raw CSVs and ml/ for notebooks, model artifacts, and pipeline scripts.
Data Pipeline (ml/data_pipeline.py) Ingest CSV, handle missing values, encode categorical features, scale numeric columns. Generate features.pkl and target.pkl for model training.
Model Training (ml/train_model.py) Use Scikit‑Learn (GradientBoostingRegressor) and optionally XGBoost. Perform train/validation split, cross‑validation, hyper‑parameter grid search. Serialize model to ml/model.pkl.
API (backend/app.py) FastAPI with endpoint POST /predict accepting JSON payload of car attributes. Load model.pkl at startup, run inference, return predicted price. Add health check endpoint.
Frontend (frontend/src/) Vite + React with Tailwind‑CSS for premium UI (dark mode, glassmorphism, micro‑animations). Form for car details, client‑side validation, and fetch to /predict. Display prediction with animated card. Optional chat widget powered by OpenAI API for price explanation.
Authentication (backend/auth.py & frontend login page) JWT‑based auth, register/login, store user profiles. Allow users to save past predictions.
CI/CD (.github/workflows/ci.yml) Run lint, unit tests for backend (pytest), frontend (npm test). Build Docker images and push to registry. Deploy to chosen cloud via GitHub Actions.
Documentation (README.md) Setup instructions, dataset acquisition, training, running locally, deployment steps. Verification Plan Automated Tests Backend unit tests (tests/test_api.py): Verify /predict returns numeric price, handles validation errors. Model tests (tests/test_model.py): Ensure training script produces a model file and that predictions are within reasonable range. Frontend component tests (src/tests/Form.test.js): Form validation and API call handling. CI pipeline: GitHub Actions will run all tests on push. Manual Verification Data pipeline: Run python ml/data_pipeline.py and confirm features.pkl created without errors. UI: Run npm run dev in frontend/; fill form with sample car data, ensure prediction card displays a price with animation. Authentication flow: Register a new user, login, save a prediction, verify it appears in profile. Deployment: Deploy to chosen cloud, access public URL, perform end‑to‑end test.
